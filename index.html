Tags HTML Atuais (Incluindo HTML5)

Esta seção detalha as principais tags HTML, seus usos, atributos e exemplos práticos.

<div> - Elemento de Divisão

Descrição

O elemento <div> é um contêiner genérico de nível de bloco para conteúdo. Ele não representa nada por si só e é usado principalmente para agrupar elementos para fins de estilização (com CSS) ou manipulação (com JavaScript).

Principais Atributos

  • id: Identificador único para o elemento.
  • class: Uma ou mais classes de CSS para estilização.
  • style: Estilos CSS inline (não recomendado para folhas de estilo grandes).
  • data-*: Atributos de dados personalizados.

Exemplo de Uso

<div class="container">
    <h1>Título da Seção</h1>
    <p>Este é um parágrafo dentro da div.</p>
</div>

<style>
.container {
    background-color: #f0f0f0;
    padding: 20px;
    border: 1px solid #ccc;
    margin-bottom: 10px;
}
</style>
<img> - Elemento de Imagem

Descrição

O elemento <img> incorpora uma imagem no documento. É um elemento vazio, o que significa que não tem tag de fechamento.

Principais Atributos

  • src (obrigatório): O caminho para o arquivo da imagem.
  • alt (obrigatório): Texto alternativo para a imagem, usado por leitores de tela e exibido se a imagem não puder ser carregada. Essencial para acessibilidade.
  • width: Largura da imagem em pixels.
  • height: Altura da imagem em pixels.
  • loading: Define como a imagem deve ser carregada (lazy para carregamento adiado, eager para carregamento imediato).

Exemplo de Uso

<img src="caminho/para/imagem.jpg" alt="Descrição da imagem" width="300" height="200">

<picture>
    <source srcset="caminho/para/imagem-grande.webp" media="(min-width: 800px)">
    <source srcset="caminho/para/imagem-media.webp" media="(min-width: 400px)">
    <img src="caminho/para/imagem-pequena.jpg" alt="Descrição responsiva da imagem">
</picture>
<form> - Elemento de Formulário

Descrição

O elemento <form> representa uma seção de um documento que contém controles interativos para o envio de informações a um servidor web.

Principais Atributos

  • action: A URL para onde os dados do formulário serão enviados.
  • method: O método HTTP a ser usado ao enviar os dados (GET ou POST).
  • autocomplete: Ativa ou desativa o preenchimento automático para o formulário.
  • target: Onde exibir a resposta após o envio do formulário (ex: _blank para nova aba).

Exemplo de Uso

<form action="/submit-form" method="post">
    <label for="username">Usuário:</label><br>
    <input type="text" id="username" name="username" required><br><br>

    <label for="email">Email:</label><br>
    <input type="email" id="email" name="email" placeholder="seuemail@exemplo.com"><br><br>

    <input type="submit" value="Enviar">
</form>

Propriedades CSS

Esta seção lista as propriedades CSS mais comuns, seus valores possíveis e exemplos de aplicação.

display - Controla o tipo de caixa de exibição

Descrição

A propriedade display especifica o comportamento de exibição de um elemento. É fundamental para o layout da página.

Valores Possíveis

  • block: Ocupa a largura total disponível e inicia uma nova linha (ex: <p>, <div>).
  • inline: Ocupa apenas a largura necessária, não inicia uma nova linha (ex: <span>, <a>).
  • inline-block: Comporta-se como inline, mas aceita width, height, margin e padding como um block.
  • flex: Estabelece um contexto de formatação flexível para seus filhos diretos.
  • grid: Estabelece um contexto de formatação de grade para seus filhos diretos.
  • none: O elemento não é exibido (remove-o do fluxo do documento).
  • e muitos outros (table, table-cell, list-item, etc.)

Exemplos de Uso

.block-element {
    display: block;
    width: 100px; /* Ignorado por inline */
}

.inline-element {
    display: inline;
    padding: 5px; /* Aceito, mas width/height ignorados */
}

.inline-block-element {
    display: inline-block;
    width: 150px;
    height: 50px;
    margin: 10px;
}

.flex-container {
    display: flex;
    justify-content: space-between;
}
color - Define a cor do texto

Descrição

A propriedade color define a cor de primeiro plano do conteúdo textual de um elemento e de quaisquer decorações de texto.

Valores Possíveis

  • Nomes de cores: red, blue, green, black, white, etc.
  • Hexadecimal: #RRGGBB (ex: #FF0000 para vermelho), #RGB (ex: #F00 para vermelho).
  • RGB/RGBA: rgb(R, G, B) (ex: rgb(255, 0, 0)), rgba(R, G, B, A) (ex: rgba(255, 0, 0, 0.5) para vermelho com 50% de opacidade).
  • HSL/HSLA: hsl(H, S%, L%), hsla(H, S%, L%, A).
  • Palavras-chave do sistema: currentColor, transparent.

Exemplos de Uso

p {
    color: blue; /* Nome da cor */
}

h1 {
    color: #336699; /* Hexadecimal */
}

.warning {
    color: rgb(255, 99, 71); /* RGB */
}

.faded-text {
    color: rgba(0, 0, 0, 0.7); /* RGBA com opacidade */
}
display: flex - Modelo de layout Flexbox

Descrição

Quando display é definido como flex em um contêiner, ele se torna um flex container, e seus filhos diretos se tornam flex items. Isso ativa o módulo Flexbox, um modelo de layout unidimensional que permite organizar itens em linhas ou colunas de forma eficiente, distribuindo o espaço.

Propriedades Principais do Flex Container

  • flex-direction: (row, row-reverse, column, column-reverse) Define a direção dos itens flex.
  • justify-content: (flex-start, flex-end, center, space-between, space-around, space-evenly) Alinha os itens ao longo do eixo principal.
  • align-items: (flex-start, flex-end, center, baseline, stretch) Alinha os itens ao longo do eixo transversal.
  • flex-wrap: (nowrap, wrap, wrap-reverse) Controla se os itens devem quebrar para a próxima linha.
  • gap: Define o espaçamento entre as linhas e colunas (substitui margin em alguns casos).

Exemplo de Uso

.flex-container {
    display: flex;
    flex-direction: row; /* Itens em linha */
    justify-content: space-around; /* Espaço igual entre e ao redor dos itens */
    align-items: center; /* Centraliza verticalmente */
    gap: 10px; /* Espaço de 10px entre os itens */
    height: 100px;
    border: 1px solid var(--primary-color);
}

.flex-item {
    background-color: var(--secondary-color);
    color: white;
    padding: 15px;
    border-radius: 5px;
}
<div class="flex-container">
    <div class="flex-item">Item 1</div>
    <div class="flex-item">Item 2</div>
    <div class="flex-item">Item 3</div>
</div>

Seletores CSS, Pseudo-classes e Pseudo-elementos

Os seletores CSS são padrões usados para selecionar os elementos HTML que você deseja estilizar. Pseudo-classes e pseudo-elementos fornecem maneiras de estilizar elementos com base em seu estado ou em partes específicas de um elemento, respectivamente.

Seletores Básicos

  • Seletor de Tipo: Seleciona todos os elementos de um determinado tipo (ex: p { color: blue; }).
  • Seletor de Classe: Seleciona elementos com um atributo class específico (ex: .minha-classe { font-weight: bold; }).
  • Seletor de ID: Seleciona um elemento com um atributo id específico e único (ex: #meu-id { font-size: 20px; }).
  • Seletor Universal: Seleciona todos os elementos (ex: * { margin: 0; padding: 0; }).

Seletores Combinadores

  • Descendente: Seleciona um elemento que é descendente de outro (ex: div p { color: green; }).
  • Filho Direto: Seleciona um elemento que é filho direto de outro (ex: ul > li { list-style: none; }).
  • Irmão Adjacente: Seleciona um elemento que é o próximo irmão do primeiro elemento especificado (ex: h1 + p { margin-top: 0; }).
  • Irmão Geral: Seleciona um elemento que é irmão do primeiro elemento especificado (ex: h1 ~ p { text-indent: 20px; }).

Pseudo-classes

Usadas para selecionar elementos com base em um estado específico, como quando o mouse está sobre eles, ou se são o primeiro filho, etc.

  • :hover: Elemento sob o cursor do mouse.
  • :active: Elemento sendo ativado (clicado).
  • :focus: Elemento que tem o foco (ex: input de formulário).
  • :nth-child(n): Seleciona o n-ésimo filho de seu pai.
  • :first-child: Primeiro filho de seu pai.
  • :last-child: Último filho de seu pai.
  • :not(): Seleciona elementos que não correspondem ao seletor dentro dos parênteses.
  • :checked: Elementos de formulário (checkbox, radio) que estão marcados.
  • :valid, :invalid: Campos de formulário com conteúdo válido/inválido.

Pseudo-elementos

Usados para estilizar uma parte específica de um elemento ou para inserir conteúdo antes ou depois do conteúdo real de um elemento.

  • ::before: Insere conteúdo antes do conteúdo de um elemento.
  • ::after: Insere conteúdo depois do conteúdo de um elemento.
  • ::first-line: Estiliza a primeira linha de um bloco de texto.
  • ::first-letter: Estiliza a primeira letra de um bloco de texto.
  • ::selection: Estiliza a parte de um elemento que foi selecionada pelo usuário.
  • ::placeholder: Estiliza o texto de espaço reservado em campos de entrada.

Exemplos

a:hover {
    color: var(--primary-color);
}

input:focus {
    border-color: var(--secondary-color);
    box-shadow: 0 0 5px rgba(0, 123, 255, 0.5);
}

li:nth-child(even) {
    background-color: #f0f0f0; /* Fundo diferente para itens pares da lista */
}

p::first-letter {
    font-size: 2em;
    font-weight: bold;
    color: var(--primary-color);
}

.icon::before {
    content: "➡️ "; /* Insere um ícone antes do texto */
}

Considerações sobre Compatibilidade e Melhores Práticas

Desenvolver para a web moderna exige atenção à compatibilidade e à adoção de melhores práticas para garantir que seus sites funcionem bem em diversos dispositivos e navegadores.

Compatibilidade entre Navegadores (Cross-Browser Compatibility)

  • Teste em Múltiplos Navegadores: Sempre teste seu código em Chrome, Firefox, Safari e Edge (e suas versões móveis) para identificar inconsistências.
  • Prefixos de Vendor: Embora menos comuns hoje com a padronização de muitas propriedades, algumas propriedades CSS experimentais ou mais recentes ainda podem precisar de prefixos como -webkit-, -moz-, -ms-, -o-. Ferramentas como o Autoprefixer podem ajudar nisso.
  • Feature Detection (Detecção de Recursos): Em vez de detectar o navegador, detecte se um recurso CSS ou JavaScript específico é suportado. Bibliotecas como Modernizr podem auxiliar, ou você pode fazer isso manualmente com JavaScript.
  • Graceful Degradation e Progressive Enhancement:
    • **Graceful Degradation:** Construa para navegadores modernos primeiro, e então forneça uma experiência básica para navegadores mais antigos.
    • **Progressive Enhancement:** Comece com uma experiência básica que funciona em todos os navegadores, e então adicione camadas de funcionalidades e estilização para navegadores mais capazes.

Melhores Práticas de HTML e CSS

  • Semântica HTML: Use as tags HTML de acordo com seu significado (ex: <header>, <nav>, <main>, <article>, <section>, <aside>, <footer>). Isso melhora a acessibilidade e o SEO.
  • Acessibilidade (ARIA): Utilize atributos ARIA (Accessible Rich Internet Applications) quando o HTML semântico não for suficiente para descrever a função ou o estado de um elemento para tecnologias assistivas.
  • CSS Modular: Organize seu CSS em módulos lógicos (ex: BEM, OOCSS, SMACSS) para facilitar a manutenção e escalabilidade.
  • Nomenclatura Consistente: Use convenções de nomenclatura claras e consistentes para classes e IDs.
  • Otimização de Performance:
    • Minimize e comprima seus arquivos HTML, CSS e JavaScript.
    • Otimize imagens.
    • Use carregamento assíncrono para scripts e folhas de estilo não críticos.
  • Responsividade (Mobile-First): Projete e desenvolva primeiro para telas pequenas, e então expanda para telas maiores usando media queries. Isso garante que a experiência mobile seja priorizada e eficiente.
  • Validação de Código: Valide seu HTML e CSS usando os validadores do W3C para garantir que seu código esteja livre de erros e siga as especificações.
